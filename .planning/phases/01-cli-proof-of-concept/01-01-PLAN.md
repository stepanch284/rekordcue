DjmdContent (ORM row from db.get_content()):
  - content.ID          # VARCHAR(255), numeric string
  - content.Title       # track title
  - content.BPM         # int, BPM * 100 (e.g. 17400 = 174.0)
  - content.Length       # int, duration in SECONDS
  - content.UUID        # VARCHAR(255), UUID string
  - content.AnalysisDataPath  # relative path to ANLZ folder

pyrekordbox.AnlzFile:
  - anlz = AnlzFile.parse_file(dat_path)
  - anlz.tag_types      # @property -> List[str] of tag codes ('PWAV', 'PQTZ', ...)
  - anlz.tags           # raw list of tag objects
  - anlz.get_tag('PWAV')  # returns tag object
  - NEVER call len(anlz) or anlz.keys() — infinite recursion bug in 0.4.4

PWAV tag:
  - pwav.get() -> (amplitudes_array, unknown_array)   # both numpy shape (400,) dtype int8
  - Always exactly 400 samples regardless of track length
  - Index i -> time: t_seconds = (i / 400) * track_length_s

PQTZ tag:
  - pqtz.times   # numpy array of beat times in SECONDS (not ms!)
  - pqtz.beats   # numpy array: 1, 2, 3, 4 (beat number within bar)
  - pqtz.bpms    # numpy array of BPM at each beat

DjmdCue schema (Rekordbox 7.2.3 verified):
  - Kind: 0=memory cue, 1=hot cue A, 2=B, 3=C, ... 8=H
  - NO "Number" column exists
  - Color: 255 for hot cues, -1 for memory cues
  - ColorTableIndex: 22 for default hot cue, 0 for memory cues
  - ID: VARCHAR(255) containing numeric string
  - created_at / updated_at: DATETIME, NOT NULL, auto-set by StatsFull mixin

  <name>Task 3: Create detect.py — energy onset detection from PWAV</name>

  <action>
Create `detect.py` in the project root with one function:

`detect_first_onset(amplitudes: np.ndarray, bar_times_s: np.ndarray, track_length_s: float, pwav_len: int = 400) -> tuple[float, int]`:

Implementation:
1. Convert bar start times to PWAV indices: `bar_indices = np.floor(bar_times_s / track_length_s * pwav_len).astype(int)`. Clip to valid range `[0, pwav_len-1]`.
2. Compute energy per bar: for each bar i, compute mean amplitude of `amplitudes[bar_indices[i]:bar_indices[i+1]]` (use `pwav_len` as end bound for the last bar).
3. Compute threshold: `mean + 0.5 * stdev` of bar_energies (exclude bars with zero energy from stats to avoid silence skew).
4. Find the first bar at index >= 4 (skip intro) where bar_energy >= threshold.
5. If found, return `(bar_times_s[onset_bar_index], onset_bar_index)` — the time in seconds and bar number.
6. Fallback: if no bar exceeds threshold, return `(bar_times_s[min(4, n_bars-1)], min(4, n_bars-1))`.

The function returns a tuple of (onset_time_seconds, bar_index). The caller multiplies seconds by 1000 for InMsec.

Print the detection details: threshold value, bar index, time in seconds and milliseconds.

Import `numpy as np`.
  </action>
